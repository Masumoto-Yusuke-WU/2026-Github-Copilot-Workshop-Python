# Pomodoro Timer 実装計画

## フェーズ 1: 土台作り

**目標**: ブラウザでページが開き、モックに近い見た目が確認できる状態にする

実装内容:
- Flask で `/` にアクセスすると `index.html` を返す最小構成
- `templates/index.html` と `static/css/style.css` の作成
- ヘッダー、タイマー表示領域（25:00 の静的表示）、ボタン、進捗カードの HTML 構造
- モックに近い配色・レイアウトの CSS
- SVG による円形プログレスの静的表示

完了基準: `flask run` でページが開き、UIモックに近い画面が表示される

---

## フェーズ 2: タイマーの核心ロジック実装

**目標**: 画面操作なしでタイマーロジックが正しく動作するテストが通る状態にする

実装内容:
- `timer-engine.js` の実装
  - `TimerState` オブジェクトの定義
  - `start()`, `pause()`, `resume()`, `reset()` の実装
  - 終了予定時刻からの残り時間再計算
  - セッション完了判定
- `session-policy.js` の実装
  - 次モードが `short_break` か `long_break` かの判定
- Python 側のユニットテスト用に同等ロジックが必要な場合は `tests/` にも追加

完了基準: ロジックのユニットテストが通る（`pytest` または Jest）

---

## フェーズ 3: タイマーの動作をブラウザで確認できる状態にする

**目標**: 開始・一時停止・リセットがブラウザで動作し、残り時間と円形プログレスが更新される

実装内容:
- `app.js` でフェーズ 2 のロジックを呼び出す配線
- `setInterval` による 1 秒ごとの再描画トリガー
- 残り時間の MM:SS 表示更新
- SVG の `stroke-dashoffset` による円形プログレス更新
- ボタンの状態制御（開始中は「一時停止」に切替など）
- セッション完了時のモード自動遷移

完了基準: 開始・一時停止・再開・リセット・モード遷移がブラウザで目視確認できる

---

## フェーズ 4: 進捗管理とデータ永続化

**目標**: ページを閉じても今日の進捗と設定が保持される

実装内容:
- `progress-service.js` の実装
  - 完了回数の更新
  - 累積集中時間の更新
  - 日付変更時のリセット判定
- `repositories.js` の実装
  - `SettingsRepository`（localStorage 実装）
  - `ProgressRepository`（localStorage 実装）
- ページ読み込み時のデータ復元
- 今日の進捗カードのリアルタイム更新

完了基準: ページリロード後も進捗と設定が復元される

---

## フェーズ 5: 通知とユーザー体験の仕上げ

**目標**: セッション完了を確実に伝え、使い勝手を整える

実装内容:
- 通知音の再生（Web Audio API または audio 要素）
- ブラウザ通知 API による完了通知
- 通知許可のリクエスト処理
- ボタンデザインの状態別スタイル調整
- モードごとの配色切替（作業中・休憩中の視覚的区別）

完了基準: セッション終了時に音と通知が届き、UI の状態が分かりやすい

---

## フェーズ 6: 設定変更と品質向上

**目標**: 時間設定を変更できるようにし、テストカバレッジを整える

実装内容:
- 設定変更 UI（集中時間・短休憩・長休憩・長休憩間隔）
- 設定変更の localStorage 保存
- フェーズ 2〜4 のロジックに対するテスト補完
- Flask ルートのテスト（200 レスポンス確認）

完了基準: 時間設定を変更でき、テストが一通り通る

---

## フェーズ間の依存関係

```
フェーズ 1（土台）
    ↓
フェーズ 2（ロジック）← テスト先行で進められる
    ↓
フェーズ 3（画面連携）
    ↓
フェーズ 4（永続化）
    ↓
フェーズ 5（通知・仕上げ）
    ↓
フェーズ 6（設定・テスト補完）
```

フェーズ 1 と フェーズ 2 は並行して進められます。ロジックを先にテストで固めてから画面と繋ぐことで、後からのバグ修正コストを下げられます。
