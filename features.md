# Pomodoro Timer 機能一覧

## 1. MVP 必須機能

### 1.1 タイマー画面表示機能
- 現在モード（作業中 / 短休憩 / 長休憩）を表示する
- 残り時間を MM:SS 形式で表示する
- 円形プログレスを表示する
- 開始ボタン、リセットボタンを表示する
- 今日の完了回数と累積集中時間を表示する

### 1.2 タイマー開始機能
- 集中セッションを開始し、残り時間のカウントダウンを始める

### 1.3 一時停止・再開機能
- 動作中のタイマーを停止する
- 同じ残り時間から再開できる

### 1.4 リセット機能
- 現在のセッションを初期状態に戻す
- 現在モードの規定時間へ残り時間を復元する

### 1.5 モード切替機能
- 集中終了後に短休憩または長休憩へ自動遷移する
- 休憩終了後に再び集中モードへ戻る

### 1.6 長休憩判定機能
- 一定回数（デフォルト 4 回）の集中完了後に長休憩へ切り替える

### 1.7 残り時間再計算機能
- 終了予定時刻を基準に残り時間を再計算する
- タブ切り替えや描画遅延でも時間ずれを最小化する

### 1.8 円形プログレス更新機能
- セッション経過率に応じてリング表示をリアルタイムに更新する

### 1.9 今日の進捗表示機能
- 今日の完了回数を表示する
- 今日の累積集中時間を表示する

### 1.10 日次進捗更新機能
- 集中セッション完了時に完了回数を +1 する
- 集中セッション完了時に累積集中時間を加算する

### 1.11 日付切替リセット機能
- 日付が変わったら今日の進捗を新しい日として初期化する

### 1.12 データ保存機能
- 設定値を localStorage に保存する
- 今日の進捗を localStorage に保存する

### 1.13 データ復元機能
- ページ再読み込み時に保存済み設定を復元する
- ページ再読み込み時に保存済み進捗を復元する

### 1.14 セッション完了通知機能
- 集中または休憩の終了時に通知音を鳴らす
- ブラウザ通知 API でポップアップ通知を表示する

### 1.15 ボタン状態制御機能
- タイマーの稼働状態に応じてボタンの表示を切り替える
- 重複開始や不正操作を防ぐ

## 2. 内部的に必要な機能

### 2.1 タイマー状態管理
- 現在モード、稼働状態、残り秒数、開始時刻、終了予定時刻、完了回数を一元管理する

### 2.2 状態遷移ロジック（TimerEngine）
- 開始、一時停止、再開、リセット、セッション完了時の遷移をまとめて扱う
- DOM や localStorage に直接依存しない純粋な実装にする

### 2.3 セッションポリシー（SessionPolicy）
- 次のモードが短休憩か長休憩かを判定するルールを独立させる

### 2.4 進捗集計ロジック（ProgressService）
- 今日の完了回数と集中時間の更新ロジックを UI から分離する
- 日付変更時のリセット判定を行う

### 2.5 保存処理の抽象化（Repository）
- SettingsRepository: 設定値の保存と読込を担当する
- ProgressRepository: 進捗データの保存と読込を担当する
- 将来 SQLite に差し替えられるよう実装を分ける

### 2.6 時刻取得の抽象化（TimeProvider）
- 現在時刻取得を直接記述しない
- テストで固定時刻を注入できるようにする

### 2.7 描画処理の分離
- 状態更新関数は新しい state を返す
- 描画関数は state を受けて DOM を更新する
- ロジック単体でテストできるようにする

## 3. 将来拡張機能

### 3.1 設定変更機能
- 集中時間を変更できるようにする
- 短休憩時間を変更できるようにする
- 長休憩時間を変更できるようにする
- 長休憩までの集中回数を変更できるようにする

### 3.2 自動開始設定
- 次の集中または休憩を自動で開始するか選べるようにする

### 3.3 履歴表示機能
- 日別の完了回数を確認できるようにする
- 日別の集中時間を確認できるようにする

### 3.4 サーバー保存機能
- Flask API を追加して進捗や設定を保存する
- SQLite にデータを永続化する

### 3.5 複数端末同期機能
- 将来ログインやクラウド保存へ拡張できる基盤を用意する
